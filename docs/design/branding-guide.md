CueMarshal Brand Identity Guide

1. Logo Text & Wordmark
The logo relies on strong, balanced geometry. The wordmark needs to stand up to the boldness of the lion emblem while feeling modern, technical, and approachable.

Primary Brand Font (Logo Text): Montserrat (Bold or ExtraBold)

Why: Montserrat is a geometric sans-serif font. Its wide, confident letterforms mirror the authoritative presence of "Marshal the Lion."

Usage: The word "Cue" can be set in the deep dark navy, and "Marshal" can be set in either navy or the bright blue accent to create visual separation without needing a space (e.g., CueMarshal).

2. Color Palette
The color palette is deliberately tight and professional, reflecting a serious developer tool while using bright accents to signify action, AI automation, and the literal "cue" that starts the pipeline.

Primary Core: Deep Dark Navy

Hex: #1F2A44

Usage: This is the anchor of the brand. Use for the primary logo emblem, primary UI backgrounds (dark mode), deep navigation bars, and bold header text in light mode. It represents stability, the Git single-source-of-truth, and Marshal’s quiet authority.

Primary Accent: Bright Blue

Hex: #00A1F0

Usage: The "Conductor's Baton." Use sparingly but purposefully. It should be used for primary call-to-action (CTA) buttons, active terminal highlights, link text, and key status indicators (like a successful pipeline run orchestrated by Devin).

Neutral Backgrounds: Snow White & Owl Grey

White Hex: #FFFFFF (For high-contrast light mode backgrounds and logo negative space).

Light Grey Hex: #F4F5F7 (For subtle UI paneling, borders, and separating agent chat interfaces).

Text/Body: Charcoal Slate

Hex: #333D55

Usage: For high-legibility body copy. It is softer than pure black, harmonizing perfectly with the deep navy.

3. Typography
As a self-hosted, Git-centric platform, the typography must prioritize legibility, UI density, and code readability.

Headings & UI Elements: Inter

Weights: SemiBold (600), Bold (700)

Usage: Inter is the gold standard for modern application interfaces. It reads beautifully on screens and gives the platform a polished, SaaS-grade feel, even when self-hosted. Use for dashboard titles, agent names (e.g., Dave the Beaver, Dot the Parrot), and navigation.

Body Copy: Inter

Weights: Regular (400), Medium (500)

Usage: For long-form documentation, Dot's technical writing outputs, and general platform instructions.

Code & Terminal Blocks: JetBrains Mono (or Fira Code)

Weights: Regular (400)

Usage: Absolutely essential for a dev platform. Use this exclusively for Linton’s linting outputs, Dave’s code generation blocks, Git commit hashes, and log files. The ligatures in these fonts make reading code much easier for human developers collaborating with the agents.
