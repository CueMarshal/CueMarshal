# Conductor Database

PostgreSQL database layer using Drizzle ORM. Manages schema definitions, client setup, and migrations.

## Files

| File | Description |
|------|-------------|
| `schema.ts` | Drizzle ORM schema — defines tables for tasks, projects, chat sessions, chat messages, and cost records |
| `client.ts` | Database client setup — creates a connection pool and exports the Drizzle `db` instance and connection test utility |
| `migrate.ts` | Standalone migration runner script that applies Drizzle migrations from the `migrations/` folder |

## Directories

| Directory | Description |
|-----------|-------------|
| `migrations/` | SQL migration files generated by Drizzle Kit |

## Purpose

This module owns the persistence layer for the Conductor. All task state, project metadata, chat history, and LLM cost records are stored in PostgreSQL and accessed through the typed Drizzle ORM client.
